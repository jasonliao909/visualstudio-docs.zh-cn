# <a name="ca2223-members-should-differ-by-more-than-return-type"></a>CA2223:成员不应只是返回类型不同

|项|“值”|
|-|-|
|RuleId|CA2223|
|类别|Microsoft.Usage|
|重大更改|重大|

## <a name="cause"></a>原因
两个公共成员或受保护成员的签名相同，但返回类型除外。

> [!NOTE]
> 此规则已被弃用。 有关详细信息，请参阅已 [弃用的规则](fxcop-unported-deprecated-rules.md)。

## <a name="rule-description"></a>规则说明
尽管公共语言运行时允许使用返回类型来区分其他相同的成员，但此功能不在公共语言规范中，也不是 .NET 编程语言的常见功能。 如果成员仅因返回类型而不同，开发人员和开发工具可能无法正确区分它们。

## <a name="how-to-fix-violations"></a>如何解决冲突
若要解决此规则冲突，请更改成员的设计，以便它们仅基于其名称和参数类型是唯一的，或者不公开成员。

## <a name="when-to-suppress-warnings"></a>何时禁止显示警告
不禁止显示此规则发出的警告。
