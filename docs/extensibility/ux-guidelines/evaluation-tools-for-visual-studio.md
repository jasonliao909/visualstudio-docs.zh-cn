# <a name="evaluation-tools-for-visual-studio"></a>Visual Studio 的计算工具
## <a name="craftsmanship-checklist-for-visual-studio"></a>适用于Visual Studio
 使用此清单来评估视觉对象和交互详细信息的用户体验质量。

### <a name="overview"></a>概述

- 验证所有命令是否都产生反馈，告知用户其命令已执行。

- 验证所有 UI 元素和控件是否在所有主题和高对比度可见。

- 验证非活动选择和主动选择是否始终区分在标准模式高对比度模式。

- 验证焦点是否始终可见且明显。

### <a name="performance"></a>性能

- 验证在命令完成时间超过一秒时是否显示某种"忙"指示器。

- 验证命令完成时间是否超过 10 秒，是否会显示显式进度栏（ (首选) 或不确定）。

### <a name="ui-text"></a>UI 文本

- 验证所有标签都是句子或标题大小写，并且没有完全小写的文本。

    ||正确|不正确|
    |-|-------------|---------------|
    |**命令文本 (所有)**|句子大小写：<br /><br /> **目录名称：**|目录名称：|
    |**客户端 (按钮)**|标题大小写：<br /><br /> **[ 设置为默认值 ]**|SET AS DEFAULT|
    |**联机 (按钮)**|句子大小写：<br /><br /> **[ 设置为默认值 ]**||

- 验证除组标头和按钮之外的所有标签是否以冒号结尾，并位于与它们配对的控件之前。

- 验证启动 UI 以捕获用户输入结束的按钮、命令和命令链接是否以省略号 [...] **。**

  示例：

  - 对话框 **上的 [高级...]** 按钮。

  - "工具" ("选项>") "菜单下的命令选项不应获得省略号，因为启动对话本身是命令的意图。

- 验证 UI 是否不包含任何缩写，行业标准术语除外。 例如，HTML 和 TCP/IP 都不需要拼写，尽管 OOM (内存不足) PII (个人身份) 信息。

### <a name="keyboard-access"></a>键盘访问

- 验证是否有方法使用键盘完成每个任务。 通常，这是通过每个控件的键盘访问完成的，但对于一些高度可视的区域，一种解决方法（如访问代码视图）是可接受的。

- 验证是否可以通过从左到右 (从上到下的顺序按逻辑顺序按控件进行) 。 虽然这是大多数控件的最佳实践，但并非所有控件都需要此方法。 例如，验证单选按钮控件是否在具有单个制表位的组中。

- 验证所有控件是否都有标签，以及每个标签是否具有助记 (异常是否包括一些非标记控件，这些控件可能位于选项卡选项卡中的标记控件) 。

- 验证没有助记符冲突。

### <a name="fonts"></a>字体

- 验证所有字体 (、大小、颜色) 一致且保持层次结构。

- 验证所有 UI 元素是否都使用环境字体服务。  ([请参阅字体和格式设置Visual Studio) ](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)

     若要检查服务是否正在使用，请转到"工具 **">"选项>字体和颜色"。** 在"设置"下拉列表中，选择"环境字体"，将字体更改为样式上不同的字体 (例如 Harrington 或一个 Sans) ，将大小设置为 12 pt。 然后，单击“确定”。 可能需要重启 IDE，但大多数 UI 会立即更改。 即使在重启时未选取字体更改的区域也不使用环境字体。

- 验证作为服务派生的字体 (例如，粗体或放大的文本) 在环境字体大小更改时保留其相对于"普通"文本的大小和格式。

- 验证是否由于字体放大而没有剪辑 bug。 剪裁的字体可能是固定高度控件或固定高度容器的结果。

### <a name="dialogs"></a>对话框

- 验证对话框标题是否与启动对话的命令相同。

- 验证所有标准控件是否与操作系统一致：背景色是标准色，任何控件都应具有特殊的重新模板样式，使其与标准控件不同。

- 验证窗体中的边距应为 12 像素，并且应显示统一且一致。

- 验证对话在 IDE shell 或生成对话的窗口内是否居中显示。

- 如果有用，对话框应可调整大小。 对于可调整大小的对话框，请验证调整大小时，相应的控件必须调整大小，而对话框的其他部分保持不变。

- 验证可调整大小的对话框是否保留任何用户调整 (大小、位置、对话框控件的扩展等) 。

- 验证标题栏中是否没有图标。

- 验证标题栏中是否没有"最小化"和"最大化"按钮。

#### <a name="dialog-operation-buttons-vs-client-only"></a>仅 VS 客户端 (对话框操作) 

- 验证操作按钮是否按此顺序：**确定、****取消**、**应用**。

- 验证"**确定****"** 和"取消"按钮是否为标准大小：75x23 像素。

- 验证 **"确定****"** 和"取消"按钮的大小是否相等，而不考虑字符串长度。

- 如果操作按钮上的标签要求按钮比标准按钮宽，请验证相应的"取消"按钮是否大小相等。

- 验证按钮和关联的控件之间是否有 6 像素的填充。

- 验证"**确定"** 和"取消"按钮是否没有助记 (由带下划线的字母定义的访问) 。

- 验证一个按钮 (**正常"，)** 具有焦点。

- 验证 **Esc** 是否取消对话框

- 验证 **Enter** 是否执行默认按钮（如果焦点不在处理 Enter 的控件中）。

- 验证"**确定"** 和"取消"按钮是否位于对话框右下角。 在极少数情况下，可以将它们垂直堆叠在右上角。

- 验证是否仅在其他按钮在对话框内水平对齐时使用垂直配置。

### <a name="control-standards"></a>控制标准

#### <a name="general"></a>常规

- 验证在可能的情况下，是否有良好的默认值来加快用户交互，并引导用户获得安全或常见结果。

- 验证标准控件的行为方式是否相同，以便用户根据早期体验了解将发生的情况。

#### <a name="label-controls"></a>标签控件

- 验证每个控件都有一个标签，并且每个标签在视觉上是否与它的控件配对 (通常在 4-6 像素范围内) 并且与其他控件更靠近其相应的控件。

- 如果标签位于上方且水平居中，则验证标签是否与控件左边缘左对齐，以便标签的基线与输入文本的基线对齐（如果位于左侧）。

- 验证如果将多个堆叠标签和输入控件定位到控件的左侧，则标签将向左刷新且与对话框边缘的距离相等，且永远不会向右刷新且与控件的距离相等。 对应均匀分布，除非它们需要额外的间距来指示分组。

#### <a name="input-controls-text-boxes-and-combo-boxes"></a>输入控件 (文本框和组合框) 

- 验证使用默认环境字体时，文本框、组合框和按钮的显示高度是否全部为 23 像素。

- 如果使用提示文本，请验证颜色是否设置为使用 `Environment.ControlEditHintText` 颜色服务。

- 如果字段是必须按此类标识的必填字段，请验证：

  - 背景设置为 且 `Environment.ControlEditRequiredBackground` 前景设置为 `Environment.ControlEditRequiredHintText`

  - 控件中出现提示文本，显示为 **" \<Required> "**

#### <a name="button-controls"></a>按钮控件

- 验证按钮的大小是否最小为 75x23 像素，除非容纳较长的文本。

- 验证按钮的左右边距是否为 3-5 像素，以及内容的填充。

- 可以使用一个仅包含省略号 **[...]** 的小正方形按钮，而不是使用 **[浏览...]** 按钮 (或类似) 。 如果使用，请验证按钮的大小是否为 23x23。

- 如果对话框中存在多个 **[Browse...]** 按钮，请验证是否 (省略号 **的缩短** 版本 [...]) 是否全部使用。

- 验证省略号 **[...]** 按钮是否没有助记键。 如果焦点位于其旁的输入控件上，则一个选项卡会将焦点移到省略号按钮上。

- 验证启动辅助 UI （捕获更多用户输入）的按钮、命令和命令链接必须以省略号 **[...]** 结束。

#### <a name="hyperlinks"></a>超链接

- 验证超链接控件在活动时是否从不闪烁红色。 这是未使用颜色服务的指示器

- 验证使用的是 VS 颜色：

  - `Environment.ControlLinkText`

  - `Environment.ControlLinkTextHover`

  - `Environment.ControlLinkTextPressed`

- 验证超链接是否显示蓝色且无下划线，除非嵌入段落。

#### <a name="check-boxes"></a>复选框

- 如果复选框包含多行文本，请验证该框是否与第一行文本对齐，而不是在所有行中垂直居中。

- 验证复选框是否始终指示二进制选择，并且不导航用户或打开新的窗口或页面。

- 如果复选框显示与输入控件相关的选项，请验证它是否位于该控件的左侧靠左和近近，以指示其关系。

- 验证复选框是否 **从不** 用作启用对话框或页面的全部内容的方法。

#### <a name="group-boxes"></a>分组框

- 验证对话框中是否不包含对话框中包含整个对话框内容的单个组框。

- 验证每个组框中是否至少有两个控件。

- 对话框中很少出现两个以上的分组框。

- 验证没有嵌套的分组框。

### <a name="icons"></a>图标

- 当在深色主题中时，验证图标是否显示正确。

- 验证所有图标是否都基于核心概念。

- 确认每个图标都是唯一的，易于识别，并且不包含两个以上的概念 (没有状态修饰符/语言) 。

- 验证基图标在空间中居中显示。

- 验证所有图标在高对比度模式下是否显示清晰。

- 验证所用的任何颜色是否与颜色使用标准保持一致。

- 验证是否没有光晕 (边框) 图标周围。  (如果存在，则光晕应与相邻 UI) 的背景色匹配。

### <a name="touch-enabled-ui"></a>支持触摸的 UI

- 验证交互式控件是否足够大，以便能够轻松可触摸-最小 **23x23 像素** 大小

- 验证最常使用的控件大小至少为 **40x40 像素** 。

- 验证交互式控件之间至少有 **5 个像素的间距**
